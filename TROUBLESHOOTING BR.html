Guia de Sobrevivência para Arquitetos/SysAdmins Seniores - Troubleshooting

1. Redes: Conectividade Básica & DNS

PROPÓSITO(Finalidade) CMD WINDOWS(Legado/Rápido) POWERSHELL(Moderno/Objeto) BASH LINUX(Universal) NOTAS(Notas do Veterano)
Exibir Informações Básicas de IP ipconfig Get-NetIPAddress -AddressFamily IPv4 ip addr show Ponto de partida universal. No Linux moderno, esqueça o ifconfig; iproute2 (ip addr) é o padrão.
Ver Configuração Detalhada (MAC/DNS) ipconfig /all Get-NetIPConfiguration -All nmcli dev show Essencial para validar se o DHCP entregou o Gateway e os sufixos DNS corretos para a VLAN.
Libertar / Renovar DHCP ipconfig /release
ipconfig /renew		 Restart-NetAdapter -Name "Eth0" dhclient -r
dhclient		 Melhor Prática: Por vezes é mais rápido e limpo reiniciar a interface de rede inteira (toggle) do que esperar pela renegociação do DHCP.
Limpar Cache DNS ipconfig /flushdns Clear-DnsClientCache resolvectl flush-caches Crítico após migrações de servidor ou falhas de propagação. Requer elevação (Admin/Root).
Consulta e Pesquisa DNS nslookup <host> Resolve-DnsName <host> dig <host> O comando dig do Linux fornece os metadados mais precisos (TTL, NameServers). No PS, Resolve-DnsName é excelente para scripting.

2. Redes: Troubleshooting Avançado & Encaminhamento

PROPÓSITO(Finalidade) CMD WINDOWS(Legado/Rápido) POWERSHELL(Moderno/Objeto) BASH LINUX(Universal) NOTAS(Notas do Veterano)
Testar Ligação (Ping) ping -t <host> Test-Connection <host> ping -c 4 <host> O ping do Linux é infinito por defeito (use -c). O ping contínuo (-t) no Windows salva vidas ao monitorizar reinícios de routers.
Rastrear Rota (Descoberta de Caminho) tracert <host> Test-NetConnection -TraceRoute mtr <host> Dica de Sénior: O mtr no Linux (My Traceroute) é imbatível; combina ping e traceroute em tempo real no ecrã.
Verificar Portas e Sockets Abertos netstat -ano Get-NetTCPConnection ss -tulpn Identifica o que está à escuta na porta. O ss (Socket Statistics) substituiu o netstat no Linux porque é drasticamente mais rápido.
Testar Porta Remota (Tipo Telnet) N/D (Requer Cliente Telnet) Test-NetConnection <IP> -Port <Porta> nc -zv <IP> <Porta> A prova de fogo da Firewall. O Test-NetConnection (TNC) é o canivete suíço moderno do Windows. No Linux, use o netcat (nc).
Ver Tabela de Encaminhamento route print Get-NetRoute ip route show Diagnóstico obrigatório quando o servidor tem múltiplas NICs e o tráfego está a sair pelo gateway errado.

3. Sistema Operativo: CPU & Memória

PROPÓSITO(Finalidade) CMD WINDOWS(Legado/Rápido) POWERSHELL(Moderno/Objeto) BASH LINUX(Universal) NOTAS(Notas do Veterano)
Informação do Sistema & Kernel systeminfo | findstr "OS" Get-ComputerInfo uname -a / hostnamectl Primeiro passo ao entrar num servidor desconhecido: conhecer a build/kernel exata para compatibilidade.
Uso de CPU/RAM em Tempo Real tasklist Get-Process | Sort CPU top / htop A CLI do Windows é fraca para visualização em tempo real. No Linux, o htop é o melhor amigo do SysAdmin para estrangulamentos.
Verificar Memória Livre systeminfo | find "Memory" Get-CimInstance Win32_OperatingSystem free -m / free -h free -h (human readable) dá a resposta em 1 segundo. Preste atenção à coluna "available" no Linux, não apenas à "free".
Tempo de Atividade do Sistema (Uptime) systeminfo | find "Boot Time" (Get-CimInstance Win32_OperatingSystem).LastBootUpTime uptime / w Servidor lento? Verifique se reiniciou sozinho durante a noite (Crash/Atualizações) validando o uptime.
Utilizadores com Sessão Iniciada quser / qwinsta Get-RDUserSession who / w Nunca reinicie um servidor crítico sem antes verificar se outro analista tem sessão iniciada na consola a realizar manutenção.

4. Armazenamento, Discos & Processos

PROPÓSITO(Finalidade) CMD WINDOWS(Legado/Rápido) POWERSHELL(Moderno/Objeto) BASH LINUX(Universal) NOTAS(Notas do Veterano)
Uso de Espaço em Disco (Volumes) wmic logicaldisk get size,freespace Get-Volume / Get-PSDrive df -h O clássico "Disco Cheio". df -h no Linux e Get-Volume no PS são autênticos salva-vidas em incidentes Sev 1.
Encontrar Pastas Grandes dir /s /a /o-s Get-ChildItem -Recurse | Sort Length du -sh * | sort -h Disco cheio? Use du -sh * na raiz para descobrir exatamente qual pasta (geralmente logs) estourou.
Matar Processo Bloqueado taskkill /F /PID <pid> Stop-Process -Id <pid> -Force kill -9 <pid> A marreta. O sinal -9 (SIGKILL) no Linux e a flag /F no Windows matam a aplicação impiedosamente quando esta bloqueia o SO.
Verificar Estado do Serviço sc query <serviço> Get-Service <serviço> systemctl status <serviço> O systemctl status do Linux moderno é brilhante porque apresenta as últimas linhas de log de erro do serviço juntamente com o seu estado.
Reiniciar Daemon/Serviço net stop <svc> && net start <svc> Restart-Service <serviço> systemctl restart <serviço> A ação mais executada na história das TI. Reiniciar o serviço resolve 80% das falhas de aplicações.
